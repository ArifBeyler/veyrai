## PURPOSE
This document defines NON-NEGOTIABLE rules to ensure the application is accepted by Apple App Review on first submission.
All UI, UX, copy, onboarding, paywall, and App Store metadata MUST comply with these rules.

Failure to follow any rule may result in App Store rejection.

---

## CORE APP DEFINITION

- The app is an AI-assisted visual generation tool.
- The app allows users to upload their own photos and generate visual results.
- The app DOES NOT guarantee realism, accuracy, or perfect results.
- The app provides inspiration and visualization only.

Forbidden claims:
- "Perfect"
- "Guaranteed"
- "Exactly like real"
- "Undetectable"
- "100% accurate"

Allowed language:
- "Generate"
- "Create"
- "Visualize"
- "AI-assisted"
- "For inspiration"

This language must be consistent across:
- App UI
- Onboarding
- Paywall
- App Store description
- Screenshots
- App preview video

---

## ONBOARDING RULES (MANDATORY)

1. The app's purpose MUST be clear within the first 10 seconds.
2. The user must understand:
   - What the app does
   - What action they can take immediately

Onboarding must include:
- One clear explanatory sentence
- One clear primary action (e.g. upload photo)

Abstract or purely aesthetic onboarding is forbidden.

---

## USER PERMISSIONS (CAMERA / PHOTOS)

All permission requests MUST include a clear justification.

Custom permission text is REQUIRED.

Example:
"Your photo is used only to generate the visual result."

Default system permission text without explanation is NOT allowed.

---

## AI USAGE DISCLOSURE

The app must clearly communicate:
- AI is used to generate visual results
- Results may vary
- Outputs are not guaranteed to be realistic

Do NOT imply:
- Professional results
- Real-world accuracy
- Identity replication

---

## FREE USAGE & PAYWALL RULES

1. Free usage limits must be clearly stated.
2. Users must know:
   - What is free
   - How many free uses they have
   - When free usage ends

Required examples:
- "You have 1 free generation."
- "Your free generation has been used."

Paywall rules:
- Must explain what the user gets
- Must NOT use vague phrases like "Unlock everything"
- Must clearly state limits, duration, and conditions
